# Security Policy

## Supported Versions

This repository is documentation and Agent Skill content. The supported version is the current default branch.

## Reporting A Vulnerability

If you find a security-sensitive issue, please do not publish exploit details in a public issue.

Use GitHub private vulnerability reporting if it is enabled for this repository. If it is not enabled, open a minimal public issue that says a sensitive report is needed, without including secrets, exploit payloads, private tokens, or vulnerable third-party targets.

Appropriate security reports might include:

- Instructions that could cause an agent to expose secrets.
- Examples that encourage unsafe dependency installation or command execution.
- Documentation that could mislead users into trusting unofficial code as official.

General documentation mistakes, broken examples, or missing guidance can be reported through normal issues.
